# Real-Time-Ferry-Ticket-Sales-Redemption-Analytics-for-Toronto-Island-Park
Detailed guide and project requirements for the Real-Time-Ferry-Ticket-Sales-Redemption-Analytics-for-Toronto-Island-Park.

# Context:
Ferries operate year-round from Jack Layton Ferry Terminal to Centre Island, Hanlan’s Point, and Ward’s Island. Understanding near real-time passenger movement is critical for operational efficiency, staffing, and public safety.
# Problem Statement
Despite having rich customer-level data, banks often lack:Although ferry ticket sales and redemptions are recorded every 15 minutes, there is no centralized analytics system to:

• Identify peak passenger flow windows
• Track net passenger movement
• Support data-driven operational planning.
# Primary Objectives
• Analyze historical and near real-time ticket activity

• Identify peak and off-peak ferry demand periods

• Quantify passenger inflow and outflow trends.

# Secondary Objectives
• Support operational decision-making for ferry scheduling.

• Improve passenger experience during peak hours.

# Key Performance Indicators (KPIs)
• Tickets Sold per Hour
• Tickets Redeemed per Hour
• Net Passenger Movement (Sales − Redemptions)
• Peak Demand Windows
• Off-Season Utilization Index.
